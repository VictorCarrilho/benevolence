# ----------- Bibliotecas ----------- #
express
    Biblioteca para gerenciar rotas.


ts-node-dev
    Biblioteca responsavel por ficar observando alterações e reiciar o servidor.

knex
    Biblioteca responsavel por facilitar comandos para varios bancos de dados diferentes.

path
    Biblioteca responsavel por padronizar caminho de arquivos no nosso projeto de acordo com o sistema operacional utilizado

cors
    Reponsavel por controlar URL's Web que podem ou não acessar nosso WebServer



# ----------- Comandos ----------- #
npx ts-node-dev src/server.ts
    Iniciar servidor


npm run scriptsDev
    Apos configurar o script no arquivo package.json, utilizar esse "atalho" de 
    comando para iniciar o servidor
    (ts-node-dev src/server.ts)






knex migrate:latest --knexfile knexfile.ts migrate:latest
    Executar as migrations(Comandos) do banco de dados do knex
    

npm run scriptsKnex:migrate 
    Apos configurar o script no arquivo package.json, utilizar esse "atalho" de 
    comando para Executar as migrations(Comandos) do banco de dados do knex
    (knex migrate:latest --knexfile knexfile.ts migrate:latest)






# ----------- Tabelas BD ----------- #

NOME:   points
OBJETIVO: Endereço e/ou pontos de coleta de doações
CAMPOS: pointID, pointName, pointEmail, pointWhatsApp, pointImage, pointLatitude, pointLongitude
        pointCity, pointUF, pointNumber


NOME:   items
OBJETIVO: Itens de doações
CAMPOS: itemID, itemTitle, itemImage



NOME:   point_items
OBJETIVO: Itens que o Ponto de Doação pode coletar
CAMPOS: pointID, itemID



# ----------- Estrutura de Pastas ----------- #
database: Tudo relacionado ao nosso banco de dados
database -> migrations: todos os scripts de CREATE, DR0P do nosso banco de dados
database -> seeds: todos os scripts de INSERT, UPDATE, DELETE de informações pré-configuradas das tabelas do banco de dados

upload: arquivos enviados via upload
